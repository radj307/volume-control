Detailed Description

Contains methods related to audio devices, sessions, and the underlying selection mechanics responsible for providing context to actions.
You can use this object to manipulate any audio session on the system.

◆ FindSessionWithIdentifier()

ISession? VolumeControl.Audio.AudioAPI.FindSessionWithIdentifier ( string  identifier,
bool  prioritizePID = true,
StringComparison  sCompareType = StringComparison.OrdinalIgnoreCase 
)

Gets a session from Sessions by parsing identifier to determine whether to pass it to FindSessionWithID(int), FindSessionWithName(string, StringComparison), or directly comparing it to the AudioSession.ProcessIdentifier property.

Parameters
identifierThis can match any of the following properties:
  • AudioSession.PID
  • AudioSession.ProcessNameUses sCompareType
  • AudioSession.ProcessIdentifierUses sCompareType
prioritizePIDWhen true (default), process IDs are prioritized over process names, which returns more accurate results when multiple sessions have identical names but may take longer.
When false, the first process with a matching ID or name is returned.
sCompareTypeA StringComparison enum value to use when matching process names or full identifiers.
Returns
ISession if a session was found.
Returns null if nothing was found.
